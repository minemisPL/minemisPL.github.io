What is JavaScript?

For those of you who don’t know, JavaScript or JS for short is a programming language to do things in a website on the client’s side (it happens in your browser, even if you turn off the Internet connection). It can make some math, allows buttons to work, new elements to dynamically appear to your eyes, dear user.

HTML and CSS on the other hand are NOT programming languages, even if they seem to be similar. They can only show you content, for every use the same and static. CSS allows you to make some animations, some dynamic actions, but still for everyone is the same.

How have I learned (basic) JavaScript?

For past two weeks I was working for this project. A simple (to use, not to make) to-do list web app using JS. Actually I made it in two days, but I had zero knowledge about JS before (except everything I already forgot from school), so I spent most of the time just learning basics.

I used the best learning technique to learn stuff I know. As simple, as it is. Just type “[technology_name] tutorial” on YouTube and pick a few hours tutorial. It shows me how to use the technology, basic features and the most importantly what possibilities I have in this technology. A few hours course doesn’t teach me how to write in JavaScript for example, but it gives me knowledge what is in there, what are similarities and differences between the technology I learn now, and those I already know. I won’t remember all the rules, all the syntax, how to make an anonymous function, for example. But now I know there are anonymous functions and if I would need them, I know what to look for. I skip some things if I already know them or they seem obvious to me. But when I notice something, what I don’t know (like anonymous functions, which I haven’t seen before) I write an example with it myself to make sure I understand how it works and where I may need to use it.

That’s only the first step. I used this course from already known Bro Code channel. Brofist for him! And here are my “notes” from it. (I don’t like to make many notes). Usually at this point I start to make a project in that technology. (You learn the best by practice!), but this time I chose to learn some clean code and project structure first. Some examples of good and bad code. I already have written a few small projects in Java (Minecraft server plugins and tiny android apps) so I know some project structure in Java.
